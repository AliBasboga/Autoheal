🩺 autoheal.js

Plug-and-play UX auto-repair · no config · no dependencies

A · Broken Image Handling auto

These images have intentionally bad URLs — autoheal replaces them with styled placeholders.

B · Global Error Handling manual trigger

Trigger JS errors or promise rejections — autoheal catches them and shows a toast.

Check the browser console for detailed error info.

C · Offline / Online Detection auto

Use DevTools → Network tab → set "Offline" to see the banner, then restore. Or use the buttons below to simulate (fires the browser events).

D · Slow Load Detection auto on load

If page load takes > 3 s, a sticky warning toast appears automatically. To test: throttle your network in DevTools before loading this page.

window.onload timing Navigation Timing API Sticky toast until loaded

E · Copy Button for Code Blocks auto

A "Copy" button is automatically injected into every <pre> element. 

// JavaScript example
function greet(name) {
  return `Hello, ${name}! 👋`;
}

console.log(greet("World"));
# Python example
def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        yield a
        a, b = b, a + b

print(list(fibonacci(8)))

Dynamic code blocks injected after load also get the copy button (MutationObserver).
